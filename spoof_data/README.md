# HFCS Data Spoofing

This directory contains tools for creating spoofed (synthetic) versions of confidential HFCS (Household Finance and Consumption Survey) data that maintain statistical properties while protecting privacy.

## Overview

The confidential HFCS data cannot be published. This spoofing framework creates synthetic versions that:

- ✅ **Pass all validation checks** (ID relationships, country codes, weights)
- ✅ **Preserve structural relationships** (households → individuals)
- ✅ **Maintain statistical distributions** (means, medians, standard deviations)
- ✅ **Preserve key correlations** (age-income relationship)
- ✅ **Keep missingness patterns** (which variables have missing data)

## Files

### Core Scripts

1. **[validate_hfcs_coherence.py](validate_hfcs_coherence.py)** - Validation checks for data coherence
   - Ensures ID relationships are consistent
   - Checks metadata columns match
   - Validates demographic data ranges
   - Can be run on both original and spoofed data

2. **[explore_hfcs_data.py](explore_hfcs_data.py)** - Data exploration and analysis
   - Categorizes columns by type (categorical, numerical, flags)
   - Analyzes distributions and ranges
   - Suggests spoofing strategies
   - Generates exploration results JSON

3. **[generate_spoofed_hfcs.py](generate_spoofed_hfcs.py)** - Main spoofing generator
   - Creates synthetic data maintaining statistical properties
   - Implements multiple spoofing strategies per column type
   - Preserves correlations between related variables
   - Regenerates flag columns

4. **[compare_original_vs_spoofed.py](compare_original_vs_spoofed.py)** - Comparison report
   - Shows statistical differences between original and spoofed
   - Checks correlation preservation
   - Validates distribution quality

## Quick Start

### 1. Generate Spoofed Data

```bash
# Generate spoofed data with validation
python spoof_data/generate_spoofed_hfcs.py \
    [path-to-original-data] \
    spoof_data/spoofed_hfcs_output \
    --validate
```

**Arguments:**
- First arg: Input directory with original P1.csv, H1.csv, D1.csv
- Second arg: Output directory for spoofed files
- `--validate`: Run validation checks after generation
- `--seed`: Random seed for reproducibility (default: 42)

### 2. Validate Existing Data

```bash
python spoof_data/validate_hfcs_coherence.py <directory>
```

### 3. Compare Statistics

```bash
python spoof_data/compare_original_vs_spoofed.py \
    --original tests/test_macro_data/unit/sample_raw_data/hfcs/2014 \
    --spoofed spoof_data/spoofed_hfcs_output
```

## Spoofing Strategy

The generator uses different strategies based on column type:

### Preserved Columns
These are kept as-is to maintain structure:
- **IDs**: `id`, `hid`, `ID` (household and individual identifiers)
- **Country code**: `SA0100` (all records are 'FR')
- **Survey metadata**: `SA0010`, `survey`, `IM0100`
- **Survey weights**: `HW0010` (critical for statistical inference)

### Categorical Columns
Resampled from observed distributions:
- Gender (`RA0200`)
- Education level (`PA0200`)
- Employment status indicators
- Tenure status (`HB0300`)

**Method**: Random sampling from value frequency distribution

### Numerical Columns
Generated using distribution fitting:

**Strictly positive with zeros** (assets, property values):
- Two-stage model: Bernoulli for non-zero + Lognormal for positive amounts
- Example: `DA1110` (home value)

**Can be negative** (income, net wealth):
- Normal distribution
- Example: `DI2000` (total income), `DN3001` (net wealth)

**Age-Income correlation**:
- Special handling to preserve age-income relationship
- Sample income conditional on age groups

### Flag Columns
Columns starting with 'f' that indicate data quality:
- Regenerated by resampling from original flag distributions
- 60 flags in P1, 453 in H1

## Results

### Validation Status
✅ **All validation checks pass** on spoofed data:
- ID relationships consistent
- Country codes consistent
- Survey weights preserved
- Demographics valid (age 0-85)
- Household sizes preserved (1-16 individuals)

### Statistical Quality

**Well-preserved variables:**
- Age (RA0300): Mean 40.90 → 40.90 (0.0% diff)
- Survey weights (HW0010): Perfectly preserved
- Gender distribution: Nearly identical
- Income mean: 31,561 → 32,180 (2.0% diff)

**Moderately preserved variables:**
- Home value (DA1110): Mean -3.0% diff, std -22.3% diff
- Total income (DI2000): Mean +2.9% diff, std -0.5% diff

**Challenging variables:**
- Secondary properties (DA1120): Large variance due to extreme outliers
- Financial assets (DA2101): High variance in spoofed data

**Correlations:**
- Age vs Income: 0.134 → 0.121 (well preserved)
- Home Value vs Income: 0.460 → 0.005 (needs improvement)

## Output Files

## Limitations and Future Improvements

### Current Limitations

1. **Correlation between assets and income not fully preserved** in D1
   - Could implement copula-based methods
   - Could use multivariate normal for key correlated variables

2. **Extreme outliers handled differently**
   - Lognormal fitting can miss very extreme values
   - Could use mixture models or kernel density estimation

3. **D1 spoofed independently from H1/P1**
   - Ideally, D1 should be recalculated from spoofed H1/P1
   - Would require understanding ECB's derivation formulas

4. **Flag columns simplified**
   - Current approach resamples flags independently
   - Could regenerate based on spoofed data characteristics
